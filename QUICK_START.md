# 快速啟動指南

## 🚀 啟動開發伺服器

### 方法 1：使用 PowerShell 腳本（推薦）

```powershell
.\start.ps1
```

### 方法 2：手動啟動

```powershell
# 1. 設定環境變數
$env:DATABASE_URL = "file:./dev.db"

# 2. 啟動開發伺服器
npm run dev
```

### 方法 3：直接執行（如果已設定 .env）

```powershell
npm run dev
```

---

## ✅ 確認伺服器已啟動

啟動成功後，您應該會看到類似以下訊息：

```
  ▲ Next.js 14.2.18
  - Local:        http://localhost:3000
  - Ready in 2.3s
```

---

## 🌐 訪問系統

打開瀏覽器，訪問：
**http://localhost:3000**

---

## 🔧 如果無法連接

### 1. 檢查伺服器是否在運行

查看終端機是否有錯誤訊息。

### 2. 檢查端口是否被佔用

如果 3000 端口被佔用，Next.js 會自動使用下一個可用端口（如 3001）。
檢查終端機輸出確認實際使用的端口。

### 3. 常見錯誤處理

**錯誤：Prisma Client 未生成**
```powershell
npx prisma generate
```

**錯誤：資料庫連接失敗**
- 確認 `dev.db` 檔案存在（首次運行時會自動建立）
- 確認 `DATABASE_URL` 環境變數設定正確

**錯誤：端口已被佔用**
```powershell
# 在另一個終端執行，找出佔用 3000 端口的進程
netstat -ano | findstr :3000
```

---

## 📝 首次啟動注意事項

1. **資料庫會自動建立**：首次運行時，系統會在 `prisma` 目錄下建立 SQLite 資料庫檔案
2. **等待編譯完成**：首次啟動可能需要較長時間（編譯和建立資料庫）
3. **查看終端輸出**：如有錯誤，會在終端機顯示詳細訊息

---

## 🛑 停止伺服器

在終端機中按 `Ctrl + C` 停止開發伺服器。

---

如有問題，請查看終端機的錯誤訊息，或參考 `TEST_GUIDE.md` 的「常見問題排查」章節。
